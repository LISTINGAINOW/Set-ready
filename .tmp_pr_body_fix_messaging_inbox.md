## What Changed
- replaced the static inbox page with live `/api/conversations` data, loading states, and real empty/error handling
- replaced the mock conversation thread with a real client-backed thread that fetches, marks messages read, and posts replies
- added unread tracking fields/migration plus accurate unread count calculation in the conversations API
- added Resend-powered new message notifications for both new threads and reply sends
- fixed an unrelated nullable `useSearchParams()` typing issue in `app/dashboard/bookings/page.tsx` so `npm run build` can pass

## Why
- the inbox was reading from static JSON instead of the database-backed messaging API
- recipients were never emailed when a new message arrived
- unread counts were hardcoded to zero and could not reflect real conversation state

## Testing
- ran `npm run build`
- verified the messaging pages now compile against the live API routes and unread-tracking flow
